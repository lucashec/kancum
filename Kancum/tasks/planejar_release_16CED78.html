    Main Description

    No começo do projeto o time irá criar um Planejamento de Release mais abstrato, pois não será possível conhecer todos os aspectos no início. O planejamento deve abordar:

    • A quantidade e a duração dos Sprints
    • Quantas pessoas ou times deverão participar do projeto
    • O número de Releases
    • O valor a ser entregue em cada Release
    • A data de liberação dos Releases
